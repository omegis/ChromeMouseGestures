# Screenshots Guide for Chrome Web Store

This document provides guidance on creating promotional screenshots for the Chrome Web Store listing.

## Required Screenshots

Chrome Web Store requires at least **1 screenshot** and allows up to **5 screenshots**.

### Recommended Screenshot Dimensions
- **1280 x 800 pixels** (16:10 aspect ratio) - Preferred
- Alternative: 1920 x 1080 pixels (16:9 aspect ratio)

## Suggested Screenshots

### Screenshot 1: Gesture Trail in Action (REQUIRED)
**What to show:**
- Browser window with a website open
- Purple gesture trail visible showing a gesture being drawn
- Caption: "Beautiful purple trail shows your gestures in real-time"

**How to capture:**
1. Open any website
2. Right-click and draw a gesture (e.g., up-down for reload)
3. While the purple trail is visible, take a screenshot
4. Crop to 1280x800 if needed

---

### Screenshot 2: Extension Popup Menu
**What to show:**
- The extension popup with toggle switches
- Shows all 6 gestures listed
- Caption: "Simple toggle to enable/disable gestures"

**How to capture:**
1. Click the extension icon in the toolbar
2. Screenshot the popup menu
3. Use image editing to add a clean background or border

---

### Screenshot 3: Multiple Gestures Demo
**What to show:**
- Collage or diagram showing different gesture patterns:
  - Left arrow (Back)
  - Right arrow (Forward)
  - Up-Down (Reload)
  - Down-Right (Close)
  - Up-Right (Next Tab)
  - Up-Left (Previous Tab)

**How to create:**
- Use a graphic design tool (Figma, Canva, Photoshop)
- Show gesture patterns with arrows
- Add labels for each gesture
- Use purple theme (#9b59b6)

---

### Screenshot 4: Toast Notification
**What to show:**
- Website with toast notification appearing in center
- Shows gesture name (e.g., "Reload Page", "Next Tab")
- Caption: "Clear feedback for every gesture"

**How to capture:**
1. Execute a gesture
2. Quickly screenshot while toast is visible (0.8s window)
3. May need to increase timeout temporarily in code for easier capture

---

### Screenshot 5: Context Menu Feature
**What to show:**
- Context menu appearing on double right-click
- Demonstrates that normal right-click menu still works
- Caption: "Double right-click for context menu"

**How to capture:**
1. Double right-click on a webpage
2. Screenshot the context menu
3. Add annotation if needed

---

## Screenshot Editing Tips

1. **Clean Backgrounds:** Use neutral websites or clean browser tabs
2. **Add Annotations:** Use arrows, circles, or text to highlight features
3. **Consistent Branding:** Use the purple theme color (#9b59b6)
4. **High Quality:** Use PNG format for crisp images
5. **Add Captions:** Include descriptive text at the bottom

## Tools for Creating Screenshots

- **macOS:** Built-in Screenshot tool (Cmd+Shift+4)
- **Windows:** Snipping Tool or Snip & Sketch
- **Browser:** Chrome DevTools Device Toolbar for consistent sizing
- **Editing:** Preview, GIMP, Photoshop, Figma, Canva

## Chrome Web Store Requirements

✅ At least 1 screenshot (we recommend 3-5)
✅ PNG or JPEG format
✅ Max file size: 16 MB
✅ Aspect ratio: 16:9 or 16:10 recommended
✅ No borders added by Chrome Web Store
